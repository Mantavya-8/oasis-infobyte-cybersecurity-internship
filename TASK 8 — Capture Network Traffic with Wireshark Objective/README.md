# Wireshark Network Traffic Analysis

## Task 8 — Capture Network Traffic with Wireshark

## Objective
The objective of this task is to capture and analyze network traffic using Wireshark. This helps understand how data packets travel across networks and how protocols such as HTTP work.

---

## Tool Used
- Wireshark

Official Website: https://www.wireshark.org/

---

## Environment
| Component | Details |
|---|---|
| Operating System | Kali Linux |
| Virtualization | VirtualBox |
| Tool | Wireshark |
| Capture Type | HTTP Traffic Analysis |

---

## Installation

```bash
sudo apt install wireshark -y
```

---

## Steps Performed

### 1. Opened Wireshark
Wireshark was launched from the Kali Linux terminal.

### 2. Selected Network Interface
The active network adapter (Wi-Fi/Ethernet) was selected for packet capture.

### 3. Started Packet Capture
Live packet capturing was started using Wireshark.

### 4. Generated Network Traffic
Traffic was generated by:
- Visiting websites
- Searching online
- Refreshing web pages

### 5. Applied HTTP Filter
The following filter was used to display only HTTP packets:

```bash
http
```

### 6. Analyzed Packets
The captured packets were analyzed to observe:
- Source IP addresses
- Destination IP addresses
- HTTP GET requests
- Packet sizes
- Protocol details

### 7. Saved Capture File
The network traffic capture was saved as:

```bash
wireshark_capture.pcap
```

---

## Traffic Analysis

### Source and Destination IP
Observed communication between the local machine and web servers through source and destination IP addresses.

### HTTP Requests
HTTP GET requests were identified while browsing websites.

### Protocol Analysis
Protocols observed during capture:
- HTTP
- TCP
- DNS
- ARP

### Packet Size Analysis
Different packet sizes were observed depending on the type of request and response.
